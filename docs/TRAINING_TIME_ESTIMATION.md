## 수렴 속도가 진짜 변수

FPS보다 **"몇 iter에 수렴하는가"** 가 더 중요하다.

```
잘 됐을 때:  5,000 iter 이내 수렴
보통:        10,000 iter
안 됐을 때:  수렴 안 함 → 보상 재설계 필요
```

**100 iter 조기 진단법:**
```python
runner.learn(num_learning_iterations=100)
# 초반 30 iter 안에 보상 상승세 확인
# → 없으면 보상 함수 재설계 (본 훈련 전에 확인)
```

## VRAM 절감 조합 (Phase 3 기준)

| 방법 | 절감 효과 | 난이도 |
|------|---------|--------|
| Parameter Sharing | 모델 크기 1/N | 쉬움 (설계 변경) |
| FP16 Mixed Precision | ~2배 절감 | 쉬움 (코드 3줄) |
| Gradient Checkpointing | ~40% 절감 | 보통 |
| env 수 축소 (512→128) | 비례 절감 | 쉬움 |
| CPU Offload (물리) | 1~2GB 절감 | 쉬움 |

**추천 조합 (Phase 3):**
Parameter Sharing + FP16 → 로봇 3대 × 128 env ≈ 6.5GB ✅
