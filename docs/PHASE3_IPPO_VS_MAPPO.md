## 3. True CTDE MAPPO 실패 패턴 분석

### 3-1. S2/S4 즉시 충돌 (스텝 5~8)

**원인**: `rew_collision = -25` 가 MPG navigation 보상보다 약해 "어차피 충돌해도 목표 도달이 이득"인 greedy 정책 학습.

```
로봇들이 목표를 향해 직진 → 중앙에서 즉시 충돌
평균스텝 5.0~8.0 = 0.33~0.53초 내 충돌
```

### 3-2. S5 교착 (스텝 299)

**원인**: 장애물 혼합 환경에서 로봇들이 서로 충돌을 회피하다가 경로를 못 찾고 정지.
충돌 페널티가 높아질수록 교착 발생 가능성도 있으나, S5는 공간이 복잡해 추가 훈련 필요.

## 5. 오늘 달성한 것 (2026-05-18)

### 기술적 해결

1. **True CTDE 구현 완료**
   - Actor: 9-dim per-robot obs → 분산 실행 가능
   - Critic: 27-dim global state → 협력 전략 학습
   - `ippo_wrapper.py`: `get_observations()` TensorDict 단독 반환 (rsl_rl 3.x 인터페이스)

2. **per-robot credit assignment**
   - 기존: 팀 평균 보상 → 모든 로봇 같은 신호
   - 수정: `per_robot_rewards` extras → 각 로봇 개별 보상

3. **rsl_rl 3.x obs_groups 디버깅 (총 4회 오류 → 해결)**
   - TensorDict `isinstance(obs, dict)` = False 이슈
   - `(TensorDict, extras)` 튜플 vs TensorDict 단독 반환 이슈
   - `on_policy_runner.py` 소스 직접 확인으로 근본 원인 해결

### 성능 개선

- 전원도달률: **0% → 40%** (처음으로 로봇들이 목표에 도달하기 시작)
- 교착률: **80% → 20%** (MAPPO 교착 문제 해결)
- S1/S3 완벽 클리어 (100% 전원도달)
